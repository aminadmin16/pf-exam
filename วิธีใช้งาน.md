## 1) ⭐ แก้ / เพิ่มข้อสอบ (ทำใน Excel — สำคัญสุด)

ข้อสอบทั้งหมดอยู่ในไฟล์ **`questions.xlsx`** มี 3 ชีต:

| ชีต | เก็บอะไร | คอลัมน์ |
|---|---|---|
| **core** | ภาค ก | `subject` · `topic` · `year` · `predicted` · `passage` · `question` · `choice1`–`choice4` · `answer` · `explanation` · `freq` |
| **special** | ภาค ข / ภาค ค | `bank` · `topic` · `predicted` · `question` · `choice1`–`choice4` · `answer` · `explanation` · `freq` |
| **banks** | ข้อมูลคลังเฉพาะทาง | `bank` · `part` · `short` · `name` · `icon` |

**กติกาการกรอก:**
- `answer` = ใส่เลข **1–4** (ข้อที่ถูก เช่น 1 = choice1) — *(จะพิมพ์ A–D ก็ได้)*
- `predicted` = ใส่ **`yes`** ถ้าเป็นข้อเก็ง 2569 · ปล่อยว่างถ้าเป็นข้อสอบจริง
- `subject` (ชีต core) = `analytical` / `ethics` / `english` เท่านั้น
- `bank` (ชีต special) = ต้องตรงกับ id ในชีต **banks** เช่น `constitution`, `pdpa`, `local`, `procurement` …
- `freq` = อัตราการออก 1–5 (ปล่อยว่างได้ ระบบจะใส่ค่าเริ่มต้นให้)
- `passage` = ใส่เฉพาะโจทย์ที่มีบทอ่าน/ตาราง (ขึ้นบรรทัดใหม่ในเซลล์ได้ด้วย Alt+Enter)

**เพิ่มข้อสอบ** = เพิ่มแถวใหม่ในชีต core หรือ special · **แก้ข้อสอบ** = แก้ในแถวนั้น

เสร็จแล้วเลือกทำอย่างใดอย่างหนึ่ง:
```bash
npm run data        # อ่าน Excel → สร้าง questions.js + dist ใหม่ (ดูในเครื่องก่อน)
```
หรือ commit + push ขึ้น GitHub ได้เลย → **Vercel จะอ่าน Excel สร้างข้อมูลใหม่ตอน deploy ให้อัตโนมัติ**

> ⚠️ **อย่าแก้ `questions.js` ตรง ๆ** — ไฟล์นี้ถูกสร้างจาก `questions.xlsx` (แก้ไปก็จะโดนเขียนทับ)
> 🛡️ ถ้า Excel เสียหาย/กรอกผิดจนข้อมูลน้อยผิดปกติ ระบบจะ **ไม่เขียนทับข้อมูลเดิม** (กันข้อมูลหายตอน deploy)

## 3) 🚀 ขึ้นเว็บ (Deploy บน Vercel)

โปรเจกต์เชื่อมกับ GitHub: **github.com/aminadmin16/pf-exam** และตั้งค่า Vercel ไว้ใน `vercel.json` แล้ว
แค่ push โค้ดขึ้น GitHub → **Vercel จะ build + deploy ให้เองอัตโนมัติ**

```bash
git add -A
git commit -m "อัปเดตข้อสอบ / แก้เว็บ"
git push
```

หลัง deploy จะได้ 3 ลิงก์:
| ลิงก์ | คือ |
|---|---|
| `เว็บของคุณ.vercel.app` | ตัวเว็บแอป (เล่นได้เลย) |
| `…/landing` | หน้าดาวน์โหลดแอป (ปุ่มไอคอน + วิธีติดตั้ง) |
| `…/landing/PFExam.apk` | ไฟล์ APK โหลดตรง |

> เว็บใช้ **cache-first** — โหลดครั้งแรกครั้งเดียวแล้วเก็บไว้ในเครื่องผู้ใช้ เปิดครั้งต่อไปเร็วมากและใช้ host น้อยสุด

## 4) 📱 ทำไฟล์ APK (Android — เซ็นใบรับรองแล้ว)

**ต้องมี:** JDK 21 (`D:\jdk21\jdk-21.0.11+10`) + Android SDK · และไฟล์ใบเซ็น `android/app/kp-release.jks` (เก็บไว้ในเครื่อง)

**คำสั่งเดียวจบ (PowerShell):**
```powershell
$env:JAVA_HOME = "D:\jdk21\jdk-21.0.11+10"
npm run data                 # ถ้าเพิ่งแก้ข้อสอบใน Excel
npm run apk:release          # build เว็บ + คัดลอกเข้า android + build APK เซ็นแล้ว
Copy-Item "android\app\build\outputs\apk\release\app-release.apk" "landing\PFExam.apk" -Force
```
ได้ไฟล์ **`android\app\build\outputs\apk\release\app-release.apk`** (เซ็นด้วยใบรับรอง CN=PF Examination, ~3.4 MB)

> 🔑 **เก็บ `android/app/kp-release.jks` ไว้ให้ดี** (ไม่ได้ขึ้น Git) — ต้องใช้ใบเดิมตอน build เวอร์ชันอัปเดต ไม่งั้นผู้ใช้จะอัปเดตทับไม่ได้
> ℹ️ ตอนติดตั้งถ้า Google Play Protect เตือน “ไม่รู้จักนักพัฒนา” ให้กด **“ยืนยันว่าจะติดตั้ง”** (อย่ากด “รับทราบ”) → ปลอดภัย เป็นเพราะแอปยังใหม่ Google เลยยังไม่รู้จัก

## 8) 📋 เพิ่มข้อสอบตามตำแหน่ง + คำสั่งให้ AI ช่วยหา/จัดรูปแบบ

โปรเจกต์มี **2 ช่องทาง** สำหรับข้อสอบ — เลือกตามประเภทที่ต้องการ:

| ช่องทาง | ไฟล์ | ใช้เมื่อ | แสดงในแอปที่ |
|---|---|---|---|
| **A) คลังทั่วไป ภาค ก/ข/ค** | `questions.xlsx` | ข้อสอบ ก.พ. ทั่วไป / คลังกฎหมาย (`computer`, `pdpa` …) | แท็บ ภาค ก · ภาค ข/ค |
| **B) ตามตำแหน่ง** | `position-data.js` | หัวข้อที่ต้องสอบ (Tick) + แนวข้อสอบเฉพาะตำแหน่ง | แท็บ **ตามตำแหน่ง** |

> ตัวอย่าง **นักวิชาการคอมพิวเตอร์** ใช้ทั้งสองทาง: ช่องทาง B สำหรับหัวข้อสอบข้อเขียน + ข้อสอบเฉพาะตำแหน่ง · ช่องทาง A สำหรับคลัง `computer` / `pdpa` ที่ผูกไว้ในหน้ารายละเอียดตำแหน่ง

---

### 8.1) ช่องทาง A — เพิ่มใน `questions.xlsx` (คลังกฎหมาย/ภาค ก)

**ขั้นตอน**
1. เปิด `questions.xlsx` → ชีต **special** (ภาค ข/ค) หรือ **core** (ภาค ก)
2. เพิ่มแถวใหม่ตามคอลัมน์ในหัวข้อ 1 ของคู่มือนี้
3. ถ้าเป็นคลังใหม่ → เพิ่มแถวในชีต **banks** ก่อน (เช่น `bank=computer`)
4. รัน `npm run data` แล้วเปิดดูในเครื่อง

**ตัวอย่างแถว special สำหรับ พ.ร.บ.คอมพิวเตอร์**

| bank | topic | question | choice1 | choice2 | choice3 | choice4 | answer | explanation |
|---|---|---|---|---|---|---|---|---|
| computer | การเข้าถึงโดยมิชอบ | องค์ประกอบสำคัญของความผิดฐานเข้าถึงระบบคอมพิวเตอร์โดยมิชอบ คือข้อใด | … | … | … | … | 1 | ตอบ …\n\n(อธิบายเพิ่ม) |

**กติกาเฉลย (สำคัญ)**
- `answer` = **1–4** (1 = choice1)
- `explanation` ขึ้นต้นด้วย **`ตอบ`** แล้วคัดลอกข้อความตัวเลือกที่ถูก
- **ห้าม** อ้าง A/B/C/D หรือ ก/ข/ค/ง ในเฉลย — ให้อ้างข้อความตัวเลือกแทน
- เฉลยแบบละเอียด: บรรทัดแรก = ตอบ + ตัวเลือกที่ถูก · บรรทัดว่าง · ย่อหน้าอธิบายเหตุผล/มาตรา/ข้อยกเว้น

---

### 8.2) ช่องทาง B — เพิ่มตามตำแหน่ง (`position-data.js`)

ใช้สำหรับ:
- **`writtenTopics`** = รายการหัวข้อที่ต้องสอบ (แสดงเป็น **รายการ Tick** ในตาราง “การประเมินครั้งที่ 1 — สอบข้อเขียน”)
- **`questions`** = แนวข้อสอบปรนัย 4 ตัวเลือก + เฉลยละเอียด (ชุดจำลองตามตำแหน่ง ~20 ข้อ)

**รหัสตำแหน่ง (posId) ที่ใช้ในระบบ**

| posId | ชื่อตำแหน่ง |
|---|---|
| `it` | นักวิชาการคอมพิวเตอร์ |
| `finance` | นักวิชาการเงินและบัญชี |
| `edu` | นักวิชาการศึกษา |
| … | ดูรายการครบใน `app.js` → `POSITION_GROUPS` |

**ขั้นตอนนำเข้า (ไม่แก้ `position-data.js` ตรง ๆ)**
1. ให้ AI สร้างไฟล์ JSON ตามรูปแบบด้านล่าง → บันทึกเป็น `_batches/_position-import.json`
2. ตรวจสอบ (preview):
   ```bash
   node _batches/build-position-data.js _batches/_position-import.json
   ```
3. ถ้า log ไม่มี structural issues → นำเข้าจริง:
   ```bash
   node _batches/build-position-data.js _batches/_position-import.json apply
   ```
4. รัน `npm run build:web` หรือ push ขึ้น GitHub

**รูปแบบ JSON ที่ระบบรับ**

```json
{
  "data": [
    {
      "posId": "it",
      "writtenTopics": [
        "ความรู้เกี่ยวกับ พ.ร.บ.ว่าด้วยการกระทำความผิดเกี่ยวกับคอมพิวเตอร์ พ.ศ. 2550 และที่แก้ไขเพิ่มเติม (ฉบับที่ 2) พ.ศ. 2560",
        "ความรู้เกี่ยวกับ พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA)"
      ],
      "questions": [
        {
          "topic": "พ.ร.บ.คอมพิวเตอร์ 2550",
          "question": "ข้อความโจทย์…",
          "choices": ["ตัวเลือก 1", "ตัวเลือก 2", "ตัวเลือก 3", "ตัวเลือก 4"],
          "answer": 0,
          "explanation": "ตอบ ตัวเลือก 1\n\nอธิบายเหตุผล มาตรา ข้อยกเว้น…"
        }
      ]
    }
  ]
}
```

**กติกา JSON (ต่างจาก Excel เล็กน้อย)**
- `answer` = **0–3** (0 = choices[0]) — *ไม่ใช่ 1–4*
- `choices` ต้องครบ 4 ข้อ **ไม่ซ้ำกัน**
- `explanation` ต้องขึ้นต้นด้วย **`ตอบ`** และ **ห้าม** ใช้ A/B/C/D หรือ ก/ข/ค/ง
- แนะนำ **12–20 ข้อ/ตำแหน่ง** · `writtenTopics` **6–10 หัวข้อ** (อิงประกาศรับสมัครจริง)
- ส่งได้หลายตำแหน่งใน `data[]` ครั้งเดียว — ระบบ merge เข้า `position-data.js` ทั้งก้อน

คัดลอกบล็อกด้านล่างไปใส่ ChatGPT / Claude / Gemini แล้วแก้ตัวแปรใน `[…]`:

```
คุณเป็นผู้เชี่ยวชาญข้อสอบ ก.พ. ภาค ข (ความรู้เฉพาะตำแหน่ง) ของไทย

## งาน
สร้างข้อมูลสำหรับแอปฝึกสอบราชการ ตามตำแหน่ง: [ชื่อตำแหน่ง เช่น นักวิชาการคอมพิวเตอร์]
posId ในระบบ: [it]

## สิ่งที่ต้องส่ง (2 ส่วน)

### ส่วน 1 — writtenTopics (รายการหัวข้อที่ต้องสอบ / Tick list)
- อ่านจากประกาศรับสมัคร ก.พ./TOR ตำแหน่งนี้
- สรุปหัวข้อความรู้เฉพาะตำแหน่งที่ใช้สอบข้อเขียน (ประเมินครั้งที่ 1)
- ครอบคลุม: [ระบุ เช่น พ.ร.บ.คอมพิวเตอร์ 2550, PDPA 2562, พ.ร.บ.ไซเบอร์ 2562, ระเบียบ ก.พ., การพัฒนาระบบ, เครือข่าย]
- จำนวน 6–10 หัวข้อ เป็นประโยคชัดเจน มีชื่อกฎหมาย/พ.ร.บ. ครบ

### ส่วน 2 — questions (แนวข้อสอบปรนัย)
- สร้าง [15] ข้อ กระจายตาม writtenTopics
- โจทย์และตัวเลือกเป็นภาษาไทย สไตล์ข้อสอบ ก.พ. จริง (ไม่คัดลอกข้อสอบจริงทั้งข้อ)
- ตัวเลือก 4 ข้อ ไม่ซ้ำ มีตัวลวงที่สมเหตุสมผล
- answer เป็น index 0–3 (0 = ตัวเลือกแรก)
- explanation บังคับ:
  - บรรทัดแรก: "ตอบ " + ข้อความตัวเลือกที่ถูก (คัดลอกทั้งประโยค)
  - บรรทัดว่าง

### 8.5) ตัวอย่างครบวงจร — นักวิชาการคอมพิวเตอร์ (`it`)
